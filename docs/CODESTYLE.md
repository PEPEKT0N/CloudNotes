# Code Style & Conventions
Этот документ описывает базовое соглашение по оформлению кода в проекте.

## 1. Общие правила
- Отступ: **4 пробела**.
- Максимальная длина строки: **120** символов.
- Концы строк: LF ('\n').
- Файлы: UTF-8 без BOM.

## 2. Именование
- 'camelCase' - переменные, поля, параметры.
- 'PascalCase' - классы, интерфейсы, свойства, методы.
- 'UPPER_SNAKE_CASE' - константы.
- 'I' - префикс для интерфейсов обязателен.

## 3. Скобки и блоки
- Открывающая фигурная скобк '{' ставится на новой строке полсе условия/цикла
```csharp
if (condition)
{
  DoSomething();
}
```
- Всегда использовать фигуркные скобки, даже для однострочных блоков.

## 4. Комментарии 
- Комментарии должны объяснять "почему", а не "что" (если код неочевиден)

## 5. Коммит
Использовать Convensional Commits:
- feat:, fix:, docs:, style:, refactor:, test:, ci:
Пример: feat: add tag filtering to notes list

## 6. Форматирвоание и статический анализ
- Для C# используем .editorconfig + dotnet format в CI.
- Для TypeScript/React используем eslint + prettier.
- Для Markdown - markdownlint.

## Pull Request
- Каждый PR должен:
    - Иметь описание изменения и инструкцию по тестирвоанию.
    - Быть кратким (по возможности).
    - Проходить CI (сборка + тесты).
